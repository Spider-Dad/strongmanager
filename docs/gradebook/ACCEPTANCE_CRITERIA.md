### Критерии приёмки — Табель успеваемости

Функциональность наставника (/progress)
- Видимость только своих студентов (по `mapping`); попытка доступа к чужим данным невозможна.
- Отчёт показывает агрегаты и краткие списки по статусам: on_time, late, no_before_deadline, no_after_deadline.
- «Сдал с опозданием» (late) относится к категории «не вовремя» и отображается явно.
- Фильтры: по тренингу, по уроку, по статусу (через inline‑кнопки).
- Сообщения компактны; при превышении лимита — корректное разбиение.

Функциональность администратора (/progress_admin)
- Агрегаты по наставникам: число отстающих (late + no_*) и доля от общего.
- Агрегаты по урокам: массовые провалы (top‑N уроков по доле отстающих).
- Быстрые выводы: наставники/уроки группы риска подсвечены в начале выдачи.

Данные и синхронизация
- Новые листы Google (`progress_config`, `progress_overrides`) читаются, валидируются и записываются в БД.
- Существующий функционал синхронизации и уведомлений не нарушен.

Безопасность и доступ
- Админ — по `config.admin_ids`; наставник — авторизованный `Mentor.telegram_id`.
- Никаких утечек данных (в логах без персональных деталей, где не требуется).

Качество
- Тесты проходят на каждом этапе (юнит/интеграция согласно плану).
- Код интегрирован без регресса: команды, middlewares и порядок хендлеров не конфликтуют.
